1. Executive Summary
This is the full product blueprint for an AI-powered swing trading signal SaaS built specifically for the Indian stock market (NSE/BSE). The platform uses a 3-layer AI system — Rule Engine, XGBoost confidence scoring, and Google's Temporal Fusion Transformer (TFT) for deep pattern recognition — to generate explainable buy/sell signals with predicted take-profit and stop-loss levels.
No equivalent product exists in India. Competitors like Tradetron and Streak are strategy-building platforms. Generic signal apps are black-box desktop software. This product is the first white-box, AI-explainable, swing-trading signal engine for Indian equities.
1.1 What Makes This Different
Explainable AI: Confidence scores are visible to users — nobody else shows this
Swing Trading Focus: 2–10 day holds. Every competitor focuses on intraday only
White-Box: All strategy rules are visible. Users see WHY a signal fired
India-Native: XGBoost and TFT trained only on NSE/BSE data — not US or global
Predicted TP/SL: TFT predicts 5-day price range. TP and SL come from the model, not a formula

3.1 Auto-Trading (Pro Only) — How It Works
When a signal passes all 3 layers and R/R ≥ 1.5, the system places the order automatically through the user's connected broker API. No human approval needed.
Signal fires → order placed via broker OAuth token → entry confirmed → system monitors price
Price hits Take Profit → auto exit order placed → position closed
Price hits Stop Loss → auto exit order placed → position closed
Kill switch active → all positions closed immediately, no new orders until user reactivates
4. System Architecture
Three layers run in sequence. Each layer has one job. Data flows one way: Rule Engine → XGBoost → TFT → User.
4.1 Layer 1 — Rule Engine (70–80% of the system)
5 strategies scan top 200 NSE stocks every 15 minutes. Each strategy is pure if-else logic. No ML here. Fast. Cheap. Reliable. This is the trigger layer — it finds "something happened." If nothing fires, the stock is skipped entirely. XGBoost and TFT never even see it.
4.2 Layer 2 — XGBoost Confidence Gate
Every raw signal from Layer 1 gets scored 0–100% by XGBoost. It looks at 15 technical features of that stock right now. If score is below 65%, the signal is killed. This is the cheap, fast filter. Runs in milliseconds. Blocks most false signals before TFT (expensive) even runs.
4.3 Layer 3 — TFT Pattern + Prediction
Only signals that passed XGBoost reach TFT. TFT reads 90 candles of history + future known events (holidays, expiry days, earnings). It outputs a 5-day price prediction with 3 ranges (pessimistic / likely / optimistic). TP and SL are calculated from this prediction. If Risk/Reward ratio is below 1.5, signal is blocked at this stage too.
⚠️ Why 3 layers and not just TFT? Because TFT is expensive to run (GPU). Running it on every stock every 15 min = $100+/day. Rule Engine + XGBoost filter first → TFT only runs on 5–15 real candidates per day → cost drops to pennies.

7.2 Training Process
Collect 2 years daily OHLCV for top 100 stocks using yfinance (50,000 rows)
Calculate all 15 features per candle using TA-Lib
Create labels: look 2 days ahead. Price up ≥ 2% → BUY. Down ≥ 2% → SELL. Otherwise → HOLD
Time-aware split: 80% train (Jan 2023–Jun 2024), 10% val (Jul–Sep), 10% test (Oct–Dec). Never shuffle.
Train with XGBClassifier: 300 trees, max_depth 6, learning_rate 0.1. Trains in under 10 seconds on CPU.
Evaluate with precision + recall. Set threshold at 65% confidence. Below = signal blocked.
7.3 Retraining
Model retrains every Sunday night automatically on Modal. New week's data is included. Model file is saved to Supabase Storage and Railway pulls the latest version on Monday.
8. Temporal Fusion Transformer (TFT)
TFT is the deep pattern recognition and prediction layer. Built by Google. It takes 90 candles of history, future known events, and static stock info. It outputs a 5-day price prediction with confidence ranges. TP and SL are derived directly from this output.
8.1 What TFT Sees — 3 Input Types
Past Known (90 candles — 23 features each)
Open, Close, High, Low, Volume, RSI 14, MACD, MACD Signal, BB Upper, BB Lower, BB %B, EMA 20, EMA 50, ATR 14, Volume Ratio, OBV, VWAP Diff, Body %, Wick %, Day of Week, Week Number, Previous Day Return, Distance to 52-Week High/Low.
Future Known (next 5 days — things we already know)
Is Holiday (NSE calendar), Is Expiry Day (Nifty/BankNifty Thursday), Is Earnings Date (company results), Trading Days Left in Week, Is RBI/Budget Event Day. This is TFT's biggest edge over LSTM — it knows what's coming.
Static (stock identity — never changes)
Sector ID, Market Cap Category (large/mid/small), Nifty 50 Member (0/1), Nifty Bank Member (0/1), Liquidity Score, Beta vs Nifty.

8.3 Inside TFT — 4 Stages
Stage
Name
What It Does
1
Variable Selection Network
Scores all 37+ features 0–1. Ignores useless ones. Different stocks get different important features.
2
Temporal Encoder (GRU)
Reads 90 candles in order. Compresses into a 512-dim vector summarizing the entire history.
3
Multi-Head Attention
Multiple heads look at different important candles simultaneously. Finds which specific days mattered.
4
Quantile Decoder
Outputs 3 numbers per day for 5 days: 10th percentile (low), 50th (likely), 90th (high). The range = confidence.
8.4 Training
Library: pytorch-forecasting (TFT is built-in, don't code from scratch)
Data: 10 years OHLCV + all 37 proxy features calculated. ~2.5M rows across 100 stocks
Lookback: 90 candles. Prediction horizon: 5 days
Loss function: Quantile Loss (not MSE). This is what gives us the 3 prediction ranges
Training: 50 epochs, early stopping, single GPU on Modal. Takes 15–30 minutes
Retrains every Sunday night same as XGBoost
9. Take Profit & Stop Loss
TP and SL come directly from TFT's prediction. No ATR formula. No generic multiplier. Every stock gets unique TP/SL based on the model's actual predicted price range.
9.1 How It Works
TFT outputs 3 values per day for days 1 through 5: 10th percentile, 50th percentile, 90th percentile
Take Profit = highest 90th percentile value across all 5 days
Stop Loss = lowest 10th percentile value across all 5 days
Risk/Reward = (TP − Entry) / (Entry − SL). If R/R < 1.5 → signal is blocked

Update - February 5, 2026: EOD Scanner and 3-Layer Pipeline
Scanner cadence is End-of-Day (EOD). The scan runs after market close and generates candidates for next-day open entry. The 15-minute intraday runner is deprecated.
Universe selection: start from all NSE listings and apply filters (price, volume, momentum) to form the tradable universe. Default is top 50-100 liquid stocks, but this is configurable.
3-layer signal pipeline: Layer 1 = 20 deterministic strategy signals (rule engine). Layer 2 = XGBoost confirmation (confidence threshold). Layer 3 = TFT quantile forecast to derive TP/SL and enforce minimum R/R before final signal.
PKScreener is used for primary filtering and additional screeners. Strategies run on the filtered universe; XGBoost and TFT act as confirmations only.
Data layer: OHLCV caching and feature persistence. Market holiday list is maintained in config.
Operational: EOD run log and daily universe are stored and visible in the admin panel.
